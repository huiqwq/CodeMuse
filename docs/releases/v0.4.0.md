# CodeMuse v0.4.0

发布日期：2026-07-13

状态：开发完成，已通过本地验收。

## 版本目标

在 v0.3.0 只读规划与上下文能力之上，增加第一套受控代码修改闭环：模型只能提交精确局部补丁，用户看到 Diff 并明确授权后才写入；同一任务的修改可在当前进程中撤销。

## 新增功能

### apply_patch

参数：

```json
{
  "path": "src/example.ts",
  "oldText": "const value = 1;",
  "newText": "const value = 2;"
}
```

安全约束：

- 目标必须是工作区内已存在的 UTF-8 普通文本文件。
- 当前任务必须先通过 `read_file` 读取目标文件。
- `oldText` 必须唯一匹配，重复或找不到都会拒绝。
- 禁止整文件覆盖，包括只省略末尾换行的形式。
- 单文件最大 1 MB，补丁片段最大 50000 字符。
- 保留原文件 CRLF 或 LF 换行风格。
- 写入使用同目录临时文件替换，并保留文件模式。

### Diff 与授权

- 写入前生成 Unified Diff。
- Diff 仅存在于内存，确认前不会修改磁盘。
- 只有输入 `y` 或 `yes` 才授权。
- 回车、`n`、其他文字、`/cancel` 或 `/exit` 均拒绝写入。
- 用户确认后再次校验文件，防止覆盖确认期间的新修改。
- 未提供确认处理器时默认拒绝。

### 任务级撤销

- 同一任务的修改记录为一个变更集。
- 同一文件被多次修改时，保留任务开始前与最终内容。
- `/undo` 展示反向 Diff 并再次确认。
- 文件被外部修改后拒绝撤销。
- 撤销记录仅保存在当前进程，成功撤销后立即清除。

### 终端安全

模型输出和 Diff 中的 ESC、响铃等控制字符会显示为 `<0x..>`，避免项目内容或远程模型改变终端显示。

## 使用方式

真实模型模式下输入明确修改任务，例如：

```text
把 src/example.ts 中的 value 从 1 改为 2，只修改这一处。
```

Agent 读取文件并提出补丁后，检查 Diff。确认正确时输入：

```text
y
```

需要撤销：

```text
/undo
```

Mock 模式不会生成补丁或修改文件，安全写入能力通过自动测试和真实模型模式使用。

## 当前限制

- 不能创建、删除或重命名文件。
- 不能执行 Shell、构建、测试或 Git 写操作。
- 只支持精确文本片段替换，不支持 AST 重构。
- 撤销历史不跨进程保存，只保留最近一次写入任务。
- 真实模型修改效果取决于模型 Tool Calling 和代码理解能力。
- 尚未使用用户的真实 DeepSeek 或 GLM API Key 联机验收。
