# CodeMuse v0.5.0

发布日期：2026-07-13

状态：开发完成，已通过本地验收。

## 版本目标

在安全读取和修改代码之后，增加受控验证能力。模型只能执行工作区根目录 `package.json` 中允许的 npm scripts，用户确认后运行，并将真实退出码和有限输出返回 Agent。

## 新增功能

### list_scripts

读取根目录 `package.json`，返回：

- 包名。
- scripts 名称。
- scripts 完整命令内容。
- 是否允许由 CodeMuse 执行。

缺少 `package.json`、JSON 无效、scripts 类型错误或文件过大时明确失败。

### run_script

只接受：

```json
{
  "script": "test",
  "timeoutMs": 60000
}
```

不接受 Shell 字符串和额外参数。模型必须在当前任务中先调用 `list_scripts`。

允许 test、build、lint、typecheck、check 及其冒号子脚本，以及 `format:check`。拒绝 dev、start、install、prepare、deploy、publish 和 pre/post 生命周期脚本。

### 用户授权

执行前显示：

```text
Command:
  npm run test --ignore-scripts
package.json:
  "test": "node tests/run.js"
```

只有用户输入 `y` 或 `yes` 才执行，其他输入默认拒绝。

### 进程控制

- Windows 使用 `node.exe + npm-cli.js`，Unix 使用 npm。
- `shell:false` 和结构化参数。
- 双重设置 `--ignore-scripts` 与 `npm_config_ignore_scripts=true`。
- 默认超时 60 秒，最大 120 秒。
- 超时或取消时终止进程树。
- stdout/stderr 合计 80 KB 上限。
- 返回退出码、超时、输出截断和持续时间。
- 命令输出同时展示给用户并作为 Tool Result 返回模型。

### 环境变量保护

项目脚本不会继承 CodeMuse 模型 API Key。名称分段包含 API_KEY、TOKEN、SECRET、PASSWORD 或 PRIVATE_KEY的环境变量会被清除。

## package.json 使用说明

开发 CodeMuse 时，以下命令必须在 CodeMuse 源码根目录执行，因为 npm 要读取当前目录的 `package.json`：

```powershell
cd "C:\Users\Administrator\Documents\Codex\2026-07-13\u-an\CodeMuse"
npm test
npm run typecheck
npm link
```

被分析项目没有 `package.json` 时，目录扫描、文件读取、搜索和局部补丁仍可使用；只有 v0.5.0 的 npm 脚本发现、构建和测试功能不可使用。

其他成员 clone 后：

```powershell
git clone https://github.com/huiqwq/CodeMuse.git
cd CodeMuse
npm install
npm test
npm run typecheck
npm link
```

## 测试

```powershell
npm test
npm run typecheck
```

共 34 项自动测试，新增覆盖：

- npm scripts 解析和允许标记。
- 缺少 `package.json` 的错误。
- 必须先 `list_scripts`。
- 非验证脚本拒绝。
- 无授权默认拒绝且不启动进程。
- 固定 npm 参数和执行授权内容。
- 失败退出码、stdout 和 stderr。
- 超时和输出截断状态。
- 敏感环境变量清理。
- Windows npm-cli.js 结构化调用。
- ModelAgent 脚本发现、授权、执行、输出和最终响应链路。

实际 `npm run typecheck --ignore-scripts` 参数形式已在本机 PowerShell 验证。Codex 沙箱禁止测试进程再次创建子进程，因此默认进程运行器在沙箱内会返回 EPERM；进程参数和所有安全分支通过可注入运行器自动测试。

## 使用示例

真实模型模式输入：

```text
检查当前修改，然后运行类型检查和测试。不要执行开发服务器。
```

模型应先调用 `list_scripts`，再对允许脚本请求执行。检查终端展示的真实脚本内容后输入 `y`。

Mock 模式不会执行脚本。

## 当前限制

- 只支持工作区根目录 Node.js/npm 项目。
- 不执行 Monorepo 子包脚本。
- 不支持任意命令或脚本参数。
- 不自动执行 npm install。
- 最长执行 120 秒。
- Windows 进程树终止依赖系统 `taskkill`。
- 尚未使用用户的真实 DeepSeek 或 GLM API Key 联机验收。
- 自动错误修复属于下一版本。
