# CodeMuse v0.3.0

发布日期：2026-07-13

状态：开发完成，已通过本地验收。

## 版本目标

让 CodeMuse 在调用模型前先理解项目结构和当前任务，自动选择相关代码，并在可控的 Token 预算内建立上下文。v0.3.0 将 v0.2.0 的“模型自行寻找文件”升级为“本地预扫描和精选上下文 + 模型按需补充工具调用”。

## 新增功能

### 项目扫描

- 识别项目名称、项目类型、主要语言、框架和包管理器。
- 识别 `package.json`、`tsconfig.json`、README 和程序入口等关键文件。
- 最多扫描 2500 个文件和 12 层目录。
- 延续 v0.2.0 的忽略目录、敏感文件和符号链接规则。

### 任务规划

每个自然语言任务建立四步计划：

```text
1. 扫描项目并识别技术栈
2. 筛选与任务相关的代码上下文
3. 分析证据并按需调用只读工具
4. 整理结论并返回结果
```

计划状态保存在 Agent 中，可通过 `/plan` 查看。

### 上下文选择

- 从任务中提取英文关键词和常见中文意图别名。
- 按关键文件、路径命中和实际内容命中进行评分。
- 最多检查 300 个候选文本文件。
- 最多选择 12 个文件，并优先把预算分配给前 4 个相关文件。
- 单文件最大读取 256 KB，单片段最多约 1600 Tokens。
- 只将精选片段交给模型，不发送整个项目。

### Token 控制

默认上下文预算为 6000 Tokens，可配置：

```powershell
$env:CODEMUSE_CONTEXT_TOKENS="8000"
codemuse .
```

允许范围为 500 到 100000。超出预算时会截断单个文件或省略低相关文件，并在 `/context` 中显示结果。Token 为本地近似估算，不等同于服务商账单。

### CLI 命令

```text
/plan       查看最近一次任务计划
/context    查看选择的文件、Token 和裁剪信息
/scan       重新扫描当前项目
/clear      清空终端和当前任务状态
```

等待任务完成并重新出现 `codemuse>` 后，`/plan` 和 `/context` 显示最终结果。任务运行时它们只显示当前进度，`/scan` 和 `/clear` 会被拒绝；`/cancel` 和 `/exit` 会停止任务。

### 模型与 Mock

- ModelAgent 首轮请求会获得项目概览和精选代码。
- 模型仍可继续调用 `list_files`、`read_file` 和 `search_code`。
- Mock 模式真实执行扫描、读取、评分和 Token 控制，但不会伪装成模型推理。
- 系统提示将代码内容标记为不可信项目数据，不能覆盖 Agent 安全规则。

## 当前限制

- 只读，不允许创建、修改或删除文件。
- 不执行 Shell、测试或 Git 写操作。
- Token 数量采用字符启发式估算，并非模型厂商的精确 tokenizer。
- 上下文选择采用关键词与内容相关性，不是语义向量检索。
- 真实 DeepSeek、GLM Tool Calling 仍取决于所选模型服务的兼容程度。
- 尚未保存会话和项目上下文摘要。
