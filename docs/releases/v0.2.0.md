# CodeMuse v0.2.0

发布日期：2026-07-13

状态：已发布，对应提交 `07d4931`，Git 标签 `v0.2.0`。

## 版本目标

将 CodeMuse 从终端模型聊天升级为能够真实查看本地代码库的只读编程 Agent。模型可以根据自然语言任务自主调用工具，工具结果返回模型后继续分析，形成首个完整 Agent Loop。

## 新增功能

### 工作区安全

- 只允许相对路径。
- 路径规范化后验证仍位于工作区。
- 使用真实路径阻止符号链接越界。
- 忽略 `.git`、`.codemuse`、`node_modules`、`dist`、`build`、`coverage` 等目录。
- 忽略 `.env`、`.npmrc` 和私钥文件。

### 只读工具

- `list_files`：列出目录和文件，限制深度和最大数量。
- `read_file`：按行读取 UTF-8 文本，最多 200 行、1 MB。
- `search_code`：普通文本搜索，返回路径、行号、列号和预览。

### Agent Loop

```text
模型流式响应
  -> 聚合 Tool Call
  -> Registry 校验参数
  -> 执行只读工具
  -> Tool Result 返回模型
  -> 模型继续调用工具或输出最终答案
```

- 单任务最多 12 轮模型请求。
- 工具失败时将结构化错误返回模型，允许调整参数。
- 支持用户取消。
- 最终摘要显示工具调用次数。

### CLI 展示

```text
→ list_files  {"path":".","maxDepth":2}
✓ list_files  列出 29 项
→ read_file   {"path":"package.json"}
✓ read_file   读取 package.json:1-16
```

Mock 模式也会执行真实本地只读工具，但不会假装具备模型推理能力。配置 API Key 后由真实模型自主选择工具。

## 使用方法

Mock 只读演示：

```powershell
npm install
npm link
codemuse .
```

也可以直接指定其他项目：

```powershell
codemuse "D:\projects\my-app"
```

真实模型：

```powershell
$env:CODEMUSE_PROVIDER="deepseek"
$env:CODEMUSE_API_KEY="你的 API Key"
$env:CODEMUSE_BASE_URL="https://api.deepseek.com"
$env:CODEMUSE_MODEL="deepseek-chat"
node src\cli.ts .
```

## 当前限制

- 只读，不允许创建、修改或删除文件。
- 不执行 Shell、测试或 Git 写操作。
- 真实模型 Tool Calling 依赖模型服务实际兼容程度。
- 尚未保存会话和项目上下文摘要。
