# CodeMuse v0.6.0

发布日期：2026-07-15

状态：开发完成，已通过本地自动验收，等待提交和创建标签。

## 版本目标

在 v0.5.0 已能返回真实脚本结果的基础上，建立受控自动修复闭环：提取失败证据、定位相关代码、指导模型修复并重新验证，同时防止模型陷入无效循环。

## 新增功能

### 失败诊断

`FailureDiagnostics` 对失败的 `run_script` 结果进行处理：

- 识别 timeout、typecheck、test、lint、build 和 runtime。
- 从 stdout/stderr 提取关键错误。
- 提取工作区内源码路径、行号和列号。
- 生成稳定的 16 位失败指纹。
- 忽略位置变化和耗时变化，识别本质相同的错误。
- 限制诊断内容长度，避免把全部构建日志再次发送给模型。

### 自动修复闭环

真实模型收到结构化诊断后，应按顺序：

1. 读取或搜索错误相关文件。
2. 只在用户明确要求修复时提出局部补丁。
3. 等待用户确认 Diff。
4. 补丁应用后重新运行原失败脚本。
5. 根据真实退出码报告成功或继续处理。

CodeMuse 不会自动批准补丁或脚本执行。

### 停止策略

- 相同脚本产生相同失败指纹两次后停止。
- 单个任务最多应用三个修复补丁。
- 达到限制后不再向模型提供工具。
- 模型只能总结失败证据、未验证内容和人工下一步。
- ModelAgent 总轮数上限由 12 调整为 20，为正常修复链路留出空间。

### 终端反馈

终端会显示：

- 已识别的失败类别。
- 是否提取到源码位置。
- 当前修复补丁次数。
- 复测成功提示。
- 自动修复停止原因。

## 安全边界

v0.4.0 和 v0.5.0 的安全规则继续生效：

- 修改前必须读取目标文件。
- 只允许精确局部补丁。
- 每次写入展示 Diff 并请求确认。
- 每次脚本执行展示真实脚本并请求确认。
- 不运行任意 Shell、安装、发布、Git 写操作或开发服务器。
- 不向脚本传递模型 API Key。
- Mock 模式不修改文件、不执行脚本、不伪造测试成功。

## API 接入

当前继续支持：

- DeepSeek 预设。
- GLM 预设。
- 自定义 OpenAI-compatible Base URL 和模型。
- PowerShell 环境变量中的 API Key。

本版本没有使用用户真实 API Key 进行外部模型验收。Provider 配置文件、连接测试、模型切换、重试和 Token 统计属于 v0.9.0。

## package.json 要求

开发 CodeMuse 时必须在源码根目录运行：

```powershell
cd "C:\Users\Administrator\Documents\Codex\2026-07-13\u-an\CodeMuse"
npm test
npm run typecheck
npm link
```

目标项目没有 `package.json` 时仍可分析和局部修改，但不能运行验证脚本，因此不能完成自动修复复测。

## 自动测试

```powershell
npm test
npm run typecheck
npm run check
```

共 38 项自动测试，v0.6.0 新增覆盖：

- TypeScript 错误分类。
- 源码路径、行号和列号提取。
- 行号和耗时变化下的稳定失败指纹。
- 三个修复补丁上限。
- 相同失败第二次出现后的停止。
- 完整“失败→诊断→读取→补丁→复测成功”Agent Loop。
- 停止后模型不再获得工具。

脚本进程参数和安全分支使用可注入 Runner 测试；当前 Codex 沙箱不允许嵌套创建真实 npm 子进程。尚未使用真实 DeepSeek 或 GLM API Key 联机验收。

## 使用示例

真实模型模式输入：

```text
运行 typecheck，修复发现的类型错误并重新验证。
```

预期过程：

```text
list_scripts
  -> run_script(typecheck)
  -> 失败诊断
  -> read_file / search_code
  -> apply_patch + 用户确认
  -> run_script(typecheck) + 用户确认
  -> 退出码 0，或停止策略触发
```

## 当前限制

- 错误位置提取主要面向 JavaScript、TypeScript 和常见前端文本文件。
- 不直接解析 Jest、Vitest、TypeScript 或 ESLint 的专用机器格式。
- 一个任务最多三个已应用修复补丁。
- 相同失败第二次出现即停止，可能需要用户开启新任务尝试其他方向。
- 不能创建新测试文件或删除、重命名文件。
- 不支持 Monorepo 子包脚本。
- 会话尚不能在退出后恢复。
- 尚未提供 Git Status 和 Diff 审查工具。
