# CodeMuse v0.10.0

发布日期：2026-07-15

状态：开发完成，已通过本地自动验收，等待提交和创建标签。

## 版本目标

解决 v0.9.0 的两个实际使用问题：API Key 不再需要每次打开 PowerShell 都重新设置；代码分析能力增加明确、可发现的代码审查入口。v0.10.0 使用 Windows DPAPI 保存本地加密凭据，并增加 `/review`、`/review --fix` 和隔离的 `/paste` 工作流。

这不是云端用户账号系统。CodeMuse 仍是本地优先 CLI，不需要自建登录服务器、数据库或 API 中转服务。

## 一次登录与持久凭据

首次保存：

```powershell
codemuse auth login deepseek
codemuse auth login glm
```

API Key 在终端中隐藏输入。状态与删除：

```powershell
codemuse auth status
codemuse auth logout deepseek
```

默认凭据文件：

```text
C:\Users\<用户名>\.codemuse\credentials.json
```

可通过 `CODEMUSE_CREDENTIALS_PATH` 覆盖。文件只保存 DPAPI 密文和环境变量标识，不保存明文 Key。GLM 的 `glm` 与 `glm-flash` Profile 共用 `ZHIPUAI_API_KEY`，保存任一 GLM Profile 后两者都可使用。

凭据解析优先级：

```text
当前进程环境变量 > Windows DPAPI 持久凭据 > 未配置
```

环境变量继续适用于 CI、临时覆盖和不希望持久化的场景。

### Windows DPAPI 安全边界

- 密文绑定当前 Windows 用户，复制到其他账号或电脑不能直接解密。
- Key 通过固定 PowerShell 子进程的标准输入交给 DPAPI。
- Key 不进入子进程参数、环境变量、终端输出、日志或 Git。
- DPAPI 子进程只继承系统运行所需的有限环境变量。
- 凭据文件限制为普通 JSON 文件、128 KB、最多 32 项。
- 拒绝符号链接、未知字段、无效标识、无效 Base64 和后端不匹配。
- Windows 之外的平台本版本继续使用环境变量，不提供不安全的明文降级。

## 专用代码审查

### 只读审查

```text
/review
/review src/models/config.ts
```

只读模式只向模型提供 read 风险工具。工具列表和 ToolRegistry 执行入口都会拒绝 write 与 execute 工具，防止模型绕过命令意图。输出必须包含严重程度、文件与行号、触发条件、影响、实际证据和修改建议。没有明确问题时必须说明剩余测试风险。

### 审查并修复

```text
/review --fix
/review --fix src/models/config.ts
```

修复模式允许 Agent 提出局部补丁和验证脚本，但每个写入和执行仍必须由用户输入 `y` 明确确认。现有路径、Diff、脚本白名单、失败修复和停止策略全部继续生效。

## 粘贴代码片段

```text
/paste
paste> const value = 1;
paste> console.log(value);
paste> .end
```

输入 `/cancel` 可取消。限制为 40000 字符和 2000 行。片段审查不扫描本地工作区、不附加本地项目文件、不给模型任何工具。内容作为 JSON 字符串中的不可信数据发送；会话不持久化完整片段。

## CLI 入口

```text
codemuse auth login <PROFILE>
codemuse auth status
codemuse auth logout <PROFILE>

/review [PATH]
/review --fix [PATH]
/paste
```

原有 `/model`、`/usage`、`/history`、`/resume`、`/undo` 等命令保持兼容。

## 自动测试

```powershell
npm test
npm run typecheck
npm run check
```

共 71 项自动测试，新增覆盖：

- CredentialStore 只保存密文并可读取、删除。
- 凭据文件未知字段与无效密文拒绝。
- `auth login/status/logout` 不显示 API Key。
- 环境变量覆盖持久凭据，GLM 两个 Profile 共用凭据。
- `/review` 只读与修复任务构造。
- `/paste` JSON 数据隔离和长度限制。
- 只读工具列表及执行入口双重拒绝写工具。
- 片段审查不扫描本地文件、不发送本地内容、不给模型工具。
- 新命令解析。

所有 71 项测试、TypeScript 类型检查、CLI 语法检查和 Git diff 格式检查通过。

## 验收限制

- 当前 Codex 沙箱禁止 Node.js 再启动 Windows PowerShell，因此无法在本次自动环境中执行真实 DPAPI 加密往返；FakeProtector 完整存储流程已通过，需在普通 PowerShell 中最终冒烟验收。
- macOS Keychain 和 Linux Secret Service 尚未实现，非 Windows 平台继续使用环境变量。
- 还没有云端用户账号、跨设备同步、验证码、数据库或模型代理计费系统。
- `/paste` 只分析片段，不会把建议自动映射到本地文件。
- `/review` 的结论质量取决于所选模型和上下文，不能代替测试与人工复核。
- 不自动 commit、push 或发布。

## Windows 手工验收

```powershell
codemuse auth login deepseek
codemuse auth status
codemuse .
```

进入后执行：

```text
/model test deepseek
/review src/models/config.ts
/paste
```

确认重新打开 PowerShell 后无需再次输入 Key。验收使用新轮换的 API Key，不要继续使用曾发到聊天中的旧 Key。
