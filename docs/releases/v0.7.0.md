# CodeMuse v0.7.0

发布日期：2026-07-15

状态：开发完成，已通过本地自动验收，等待提交和创建标签。

## 版本目标

让 CodeMuse 在退出后仍能查看历史任务，并在工作区没有变化时恢复旧计划、上下文摘要和安全的工具活动记录。恢复不是重放旧写入，而是把经过校验的历史摘要作为下一条任务的背景，所有结论仍需重新验证。

## 新增功能

### 本地会话存储

每条自然语言任务结束后，在目标项目中保存：

```text
.codemuse/
└─ sessions/
   └─ <UUID>.json
```

记录内容：

- 任务、模型名称、模式和完成状态。
- 项目扫描、任务计划和上下文摘要。
- 工具成功或失败摘要。
- 用户授权类型、路径和批准/拒绝结果。
- 自动修复完成或停止摘要。
- 工作区检查点。

不会记录：

- 明文 API Key。
- Diff 完整内容。
- stdout/stderr 完整输出。
- 模型逐字流式回答。
- 可用于自动重放写入或命令的参数。

### /history

```text
/history
```

显示当前工作区最近 10 条会话，包括：

- 8 位会话 ID 前缀。
- 保存时间。
- 完成、失败、取消或停止状态。
- 原任务。

工作区最多保留 50 条会话，旧记录自动清理。

### /resume

恢复最新会话：

```text
/resume
```

通过 ID 前缀恢复：

```text
/resume 1a2b3c4d
```

恢复后：

- 旧计划和上下文可通过 `/plan`、`/context` 查看。
- 下一条自然语言任务会收到旧任务、状态、摘要、计划和最近 10 条活动。
- 历史内容按不可信数据处理，不能覆盖系统提示。
- Agent 会重新扫描和读取当前项目。

### 工作区变化校验

会话保存时对项目文件清单、大小和修改时间生成 SHA-256 检查点。

恢复前重新计算：

- 文件数量不同：拒绝恢复。
- 文件大小或修改时间不同：拒绝恢复。
- 项目扫描超过 2500 个文件并被截断：拒绝恢复。
- `.codemuse/` 自身不参与检查点。

拒绝恢复时应开始新任务重新扫描，避免使用过期代码上下文。

### 存储安全

- `.codemuse` 被 CodeMuse 项目扫描和上下文选择忽略。
- 会话目录 realpath 必须留在工作区内，拒绝符号链接越界。
- 单文件最大 512 KB。
- 只读取合法 UUID 文件和 schemaVersion 1。
- 对任务、摘要、路径、活动数量和恢复上下文长度做限制。
- API Key、Bearer Token 和常见 `sk-` Key 形式会被替换为 `[REDACTED]`。
- 每次恢复最多向模型提供最近 10 条、每条 400 字符活动。

## 使用方法

运行一个任务并正常结束后：

```text
codemuse> 分析项目结构
...
会话已保存：1a2b3c4d

codemuse> /history
codemuse> /resume 1a2b3c4d
codemuse> 根据上次结果继续检查入口文件
```

建议在被分析项目的 `.gitignore` 中加入：

```gitignore
.codemuse/
```

CodeMuse 不会擅自修改目标项目的 `.gitignore`。

## 当前边界

- 只在任务结束时保存，不支持进程崩溃后的任务中途恢复。
- 不恢复模型厂商内部对话 ID。
- 不自动重新执行旧补丁或旧脚本。
- `/undo` 仍只支持当前 CodeMuse 进程中的最近修改。
- 工作区任何已检测变化都会拒绝恢复，没有 `--force` 绕过。
- 检查点依据文件清单、大小和修改时间，不是所有文件内容哈希。
- 会话存储在目标项目本地，不同步到云端。
- 用户应避免把 `.codemuse/` 提交到 Git。
