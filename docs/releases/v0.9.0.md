# CodeMuse v0.9.0

发布日期：2026-07-15

状态：开发完成，已通过本地自动验收，等待提交和创建标签。

## 版本目标

在 v0.8.0 完整本地编程闭环基础上，让一个 CodeMuse CLI 进程安全管理多个模型服务：通过本机 Profile 配置 DeepSeek、GLM、OpenAI 和自定义 OpenAI-compatible Provider，在任务之间切换模型，执行最小连接测试，对临时 API 错误有限重试，并展示供应商返回的 Token 用量。

## 本机 Profile 配置

默认配置文件：

```text
C:\Users\<用户名>\.codemuse\config.json
```

可通过 `CODEMUSE_CONFIG_PATH` 覆盖。使用：

```text
codemuse> /model init
```

生成 schemaVersion 1 模板，内置 deepseek、glm、glm-flash、openai。glm 使用 glm-5.2，glm-flash 使用 glm-4.7-flash，两者共用 ZHIPUAI_API_KEY。每个 Profile 支持：

| 字段 | 说明 |
|---|---|
| `name` | CLI 切换使用的唯一名称 |
| `provider` | Provider 标识 |
| `baseUrl` | http/https API 根地址 |
| `model` | 服务商模型名称 |
| `apiKeyEnv` | 保存 API Key 的环境变量名 |
| `timeoutMs` | 可选，5000-120000 |
| `maxRetries` | 可选，0-5 |

安全限制：

- 文件最多 128 KB。
- 最多 20 个 Profile。
- 拒绝符号链接配置文件。
- 严格拒绝未知字段。
- 明确拒绝 `apiKey` 字段。
- API Key 只从当前进程环境变量读取。
- 配置模板权限在支持的平台上设置为仅当前用户。

旧的 `CODEMUSE_PROVIDER`、`CODEMUSE_API_KEY`、`CODEMUSE_BASE_URL` 和 `CODEMUSE_MODEL` 继续生效，并映射为内存中的 `environment` Profile。

## 多模型 CLI

新增命令：

```text
/model
/model list
/model use <NAME>
/model test [NAME]
/model init
/model reload
/usage
```

### 查看与切换

`/model` 和 `/model list` 显示：

- Profile 名称。
- Provider/模型。
- 是否配置对应环境变量。
- 当前活动 Profile。
- 本机配置文件路径。

不显示 API Key。

`/model use` 只能在没有任务运行时执行。ManagedAgent 在切换前保存 AgentSessionState，新建 ModelAgent/MockAgent 后恢复状态，并继续共享同一个 ToolRegistry。因此：

- 当前计划和上下文保留。
- 恢复的历史状态保留。
- 当前进程最近修改仍可 `/undo`。
- Token 用量继续累计。
- 正在运行的任务不能中途换模型。

`/model use mock` 可随时回到不联网的本地演示。

### 重新加载

修改 JSON 后执行：

```text
/model reload
```

重新校验配置和当前进程环境变量。操作系统不能把后来设置的父进程环境变量注入已经运行的 CodeMuse；如果刚设置 Key，应退出并从该 PowerShell 重新启动。

## API 连接测试

```text
/model test
/model test glm
```

连接测试使用真实 `POST /chat/completions`：

- `stream: false`。
- 简短固定提示。
- `max_tokens: 1`。
- 返回成功/失败、延迟、请求尝试次数和可用 Token usage。
- 不发送项目代码、会话上下文或工具定义。
- 可能产生极少量计费。

Mock 连接测试不访问网络。

## 超时与有限重试

默认：

```json
{
  "timeoutMs": 30000,
  "maxRetries": 2
}
```

可重试：

- 临时网络失败。
- HTTP 429。
- HTTP 500、502、503、504。

不重试：

- 400 请求错误。
- 401/403 鉴权与权限错误。
- 其他确定性 4xx。
- 已经开始读取流式正文后的错误。

系统读取 `Retry-After`，单次等待最多 5 秒；否则使用 250ms 起始的指数退避。成功重试后终端显示第几次尝试成功。达到上限时如实返回错误。

错误详情最多保留 300 字符，并删除当前 API Key。

## Token 统计

流式请求增加兼容的：

```json
{
  "stream_options": {
    "include_usage": true
  }
}
```

支持 `prompt_tokens/completion_tokens/total_tokens`，也兼容 `input_tokens/output_tokens`。

每次 API 返回 usage 时：

- ModelAgent 产生 `model-usage` 事件。
- CLI 显示本次输入、输出和总 Token。
- ManagedAgent 按模型累计。
- `/usage` 显示当前进程各模型和合计。
- SessionRecorder 只保存有限数字摘要。

不会保存提示词、模型完整回复或 API Key。Token 用量是供应商响应值，不等同于账户账单，也不计算价格。

## 使用步骤

首次准备：

```powershell
codemuse .
```

在 CodeMuse 中：

```text
/model init
/exit
```

在 PowerShell 设置需要的 Key：

```powershell
$env:DEEPSEEK_API_KEY="你的 DeepSeek API Key"
$env:ZHIPUAI_API_KEY="你的 GLM API Key"
$env:OPENAI_API_KEY="你的 OpenAI API Key"
codemuse .
```

进入后：

```text
/model list
/model test deepseek
/model use deepseek
分析当前项目的模型调用流程
/usage
/model use glm
继续检查错误处理
/model use glm-flash
```

自定义服务在配置文件中新增 Profile 和独立 `apiKeyEnv`。

当 glm-5.2 的限时免费活动结束或需要降低成本时，可执行 /model use glm-flash 切换到已验收的 glm-4.7-flash。价格和免费额度以智谱控制台为准。

## 已知限制

- OpenAI 未执行公网模型验收。
- 自定义服务必须兼容 `/chat/completions`、SSE Tool Calling 和所用请求字段。
- 不支持 OAuth、云厂商签名认证或非 Bearer Key。
- API Key 不由 CodeMuse 持久化，必须在启动进程环境中存在。
- `/model reload` 无法获得进程启动后才在其他终端设置的环境变量。
- Token 统计依赖服务商返回 usage；没有 usage 时不估算。
- Token 统计只在当前进程内累计，不计算价格。
- 重试只覆盖初始 HTTP 请求，流式正文中断后不自动重发。
- 连接测试是实际 API 请求，可能产生少量计费。
- 不支持任务运行中途切换模型。
- 不自动 commit、push 或发布。
