# CodeMuse v0.8.0

发布日期：2026-07-15

状态：开发完成，已通过本地自动验收，等待提交和创建标签。

## 版本目标

在 v0.7.0 的代码分析、局部补丁、受控验证、自动修复和会话恢复基础上，补齐普通文本文件的创建、重命名和删除，并提供只读 Git Status、Diff、当前分支和变更归属，让 Agent 在任务结束前能够给出可核对的本地变更总结。

## 新增功能

### create_file

`create_file` 在已存在的工作区目录中创建新的 UTF-8 文本文件：

- 目标路径必须不存在。
- 父目录必须已经存在。
- 最多 100 KB。
- 拒绝绝对路径、路径越界、敏感路径、忽略目录、二进制扩展名、NUL 和无效 Unicode。
- 创建前显示 Unified Diff；空文件显示明确创建清单。
- 用户确认后再次检查目标是否仍为空闲。
- 默认拒绝，只有输入 `y` 或 `yes` 才写入。

### rename_file

`rename_file` 重命名普通 UTF-8 文本文件：

- 模型必须先在当前任务调用 `read_file`。
- 原路径和目标路径都必须位于工作区。
- 目标必须不存在，父目录必须已经存在。
- 拒绝二进制、敏感、忽略和符号链接路径。
- 执行前展示原路径、目标路径和授权提示。
- 确认后重新校验原内容与目标占用。
- 使用排他目标创建语义，避免覆盖同名文件。

### delete_file

`delete_file` 删除当前任务已经读取的普通 UTF-8 文本文件：

- 最多处理 1 MB 文件。
- 拒绝二进制、敏感、忽略和符号链接路径。
- 删除前显示删除 Diff。
- 确认后重新读取，文件变化时拒绝删除。
- 删除记录进入 ChangeJournal，可通过 `/undo` 恢复。

### 混合操作撤销

ChangeJournal 现在按顺序记录：

```text
create
modify
rename
delete
```

`/undo` 会先校验任务完成后的所有路径，再显示逆向操作。批准后按相反顺序执行，因此可以恢复：

```text
重命名 src/a.ts -> src/b.ts
修改 src/b.ts
删除 src/b.ts
/undo
=> src/a.ts 恢复到任务前内容
```

单任务最多涉及 20 个文件和 40 次文件操作。撤销仍只支持当前 CodeMuse 进程中的最近一次修改任务。

### git_status

`git_status` 只读返回：

- 当前 Git 分支。
- Porcelain 状态码。
- 文件路径和重命名原路径。
- 是否发生输出截断。
- Agent 当前任务文件操作摘要。
- 每个状态项的变更归属。

归属类型：

| 类型 | 含义 |
|---|---|
| `user-existing` | 本任务首次写入前已经存在，Agent 未涉及 |
| `agent` | 本次 Agent 新产生 |
| `user-and-agent` | 任务前已有修改，Agent 又修改同一路径 |
| `user` | 基线后出现但不属于 Agent 日志 |

ToolRegistry 在当前任务首次写操作之前保存一次只读 Git 状态基线。没有写操作时，第一次 `git_status` 读取本身成为基线。

### git_diff

`git_diff` 支持：

- 默认查看未暂存 Diff。
- `staged: true` 查看已暂存 Diff。
- 通过安全的相对路径只查看一个文件。
- 未指定路径时排除 `.env`、`.codemuse`、依赖目录和构建目录。
- 10 秒超时。
- 80 KB stdout/stderr 合并输出上限。
- 明确返回退出码、超时和截断结果。

Git 子进程使用固定参数与 `shell:false`。CodeMuse 没有 add、commit、checkout、reset、push 或其他 Git 写工具。

### 最终变更总结

ModelAgent 成功结束任务时输出确定性摘要，例如：

```text
本次 Agent 文件操作：新建 1、修改 2、重命名 1；涉及 src/a.ts、src/b.ts
```

会话记录保存有限摘要，不保存完整 Diff。

## 使用方法

进入要处理的项目：

```powershell
cd "D:\projects\my-app"
codemuse .
```

使用真实模型时可以输入：

```text
codemuse> 新建 src/config.ts，定义 AppConfig 类型
codemuse> 把 src/old-name.ts 重命名为 src/new-name.ts
codemuse> 删除已经不用的 src/legacy.ts
codemuse> 查看当前 Git 状态，区分我原有的修改和你本次的修改
codemuse> 查看 src/index.ts 的 Git Diff
```

出现确认提示时先检查路径和 Diff。输入 `y` 批准，其他输入拒绝。

撤销当前进程最近一次任务修改：

```text
codemuse> /undo
```

CodeMuse 不会提交代码。任务结束后回到 PowerShell，由用户检查并提交：

```powershell
git status
git diff
git add .
git commit -m "your message"
git push origin main
```

## 安全边界

- 只支持普通 UTF-8 文本文件，不处理目录生命周期。
- 不创建缺失父目录。
- 不覆盖创建或重命名目标。
- 重命名和删除必须先读取。
- 每项文件操作单独授权，授权期间变化会被拒绝。
- Git 状态和 Diff 只读。
- 不执行任意 Shell、npm install 或 Git 写命令。
- 不保存或提交 API Key。
- 项目内容、Git 输出和恢复会话都视为不可信数据。

## 已知限制

- Mock 模式只说明能力边界，不会伪造模型工具调用；完整自然语言操作需要配置真实兼容模型。
- 未跟踪文件会出现在 Git Status，但标准 `git diff` 不显示其内容。
- Git 变更基线只在当前任务内存中存在，不随会话保存。
- `/undo` 只恢复当前进程最近一次修改任务。
- 文件创建不自动建立目录。
- 不处理目录重命名、目录删除、二进制文件和超大文本。
- Git 可执行文件必须在系统 PATH 中。
- 本版本没有自动 commit、push、分支切换或冲突解决。
- 未使用真实 DeepSeek/GLM API Key 进行外部模型验收。
